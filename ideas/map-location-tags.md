# 产品 Brief：地图定位标签与双向市场工作区

## 原始需求

求职者可以在地图上查看各区域招聘信息，招聘者可以在地图上查看各区域求职信息。地图标记可点击进入详情，支持缩放、区域聚合、电话联系、收藏和拉黑。小程序提供地图、卡片列表、我的三个 Tab。管理员可以管理发布信息并将信息打回，打回状态要在地图和卡片列表中明显展示，并可作为筛选条件。

## 事实、假设与未知

| 类型 | 内容 | 来源/验证 |
| --- | --- | --- |
| 事实 | 第一阶段只实现微信小程序，iOS 和 Android 暂不实现 | 用户已确认 |
| 事实 | Web 是管理员管理后台 | 用户已确认 |
| 事实 | 双方已有发布信息、详情、联系方式、收藏和下架能力 | 现有第一版市场需求 |
| 假设 | 地图只展示区域级位置，不公开精确坐标和楼栋地址 | 隐私和安全默认，架构阶段确认 |
| 假设 | 拉黑只对当前用户隐藏目标，不通知对方且不影响其他用户 | MVP 可逆设计 |
| 假设 | 管理员打回表示信息需要修改，发布者可看到原因并重新提交 | 参照现有身份审核状态 |
| 未知 | 地图服务、逆地理编码和正式地图密钥 | 架构阶段确认 |
| 未知 | 打回是否需要二次审核后才能重新公开 | 产品/运营确认，MVP 默认重新审核 |

## MVP

- 求职者地图 Tab：查看招聘信息标记、聚合数量、详情和联系方式。
- 招聘者地图 Tab：查看求职信息标记、聚合数量、详情和联系方式。
- 地图与卡片列表使用同一套筛选条件和信息状态。
- 地图支持缩放；低缩放级别按区域聚合，高缩放级别展开为单点。
- 标记点击进入与卡片一致的详情页面。
- 详情支持电话联系、收藏、取消收藏、拉黑和举报。
- “我的”Tab：个人资料、身份状态、我的发布、我的收藏、我的拉黑、隐私协议和基础设置。
- 管理员可以查看、打回、下架和恢复处理信息；打回原因可被发布者看到。
- 地图和卡片列表展示 `publishedAt`、发布状态和打回状态。

## 非目标

- 自动匹配、推荐算法、地图找房式路线规划。
- 精确楼栋地址、精确个人位置和实时位置共享。
- 聊天、投递、申请状态和通知。
- iOS、Android 和普通用户 Web 端。

## 隐私默认

- 地图点使用区域级坐标或聚合中心点，服务端不向对方返回原始经纬度。
- 招聘者楼栋级详细地址永不进入普通用户地图、列表或详情响应。
- 联系方式只在已登录且拥有匹配审核通过身份的详情中展示，并记录访问。
- 拉黑列表只对当前用户生效。

## 核心验收

- 两种角色都能进入对应地图和卡片列表 Tab。
- 地图与列表展示同一批有效信息，筛选和状态一致。
- 缩放时单点和聚合点正确切换，聚合点显示数量并可继续放大。
- 点击标记能进入详情，详情包含发布时间和联系方式但不泄露精确地址。
- 收藏、拉黑、举报成功/失败/重试状态可见且权限正确。
- 管理员打回后，地图和卡片显示明显状态、原因和筛选结果；信息不再作为正常公开信息展示。
- “我的”Tab 能查看和修改允许修改的个人资料，管理收藏、拉黑和设置。
